# Create A PRD

Write a structured markdown PRD to the requested file, defaulting to `PRD.md`.

## Required Sections

- Executive Summary
- Mission
- Target Users
- MVP Scope
- User Stories
- Core Architecture And Patterns
- Tools Or Features
- Technology Stack
- Security And Configuration
- API Specification when needed
- Success Criteria
- Implementation Phases
- Future Considerations
- Risks And Mitigations

## Rules

- use concrete examples
- call out assumptions explicitly
- keep scope realistic
